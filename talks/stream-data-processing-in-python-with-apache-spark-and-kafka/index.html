Stream Data processing in Python with Apache Spark and Kafka

Peter Hoffmann

Apache Spark is a computational engine for large-scale data processing. It is responsible for scheduling, distribution and monitoring applications which consist of many computational task across many worker machines on a computing cluster. Apache Kafka is publish-subscribe messaging based on a distributed, partitioned, replicated commit log service.

This talk will give an overview into Apache Spark with a focus on PySpark Streaming, the API for real-time streaming data processing. While Spark Core itself is written in Scala and runs on the JVM, PySpark exposes the Spark programming model to Python. The high-level abstraction to live input data streams are so called discretized streams or DStreams. DStreams group input streams as a series of short batch jobs. Each item in a DStream is a Resilient Distributed Dataset (RDD). RDDs are a distributed memory abstraction that lets programmers perform in-memory computations. RDDs are immutable, partitioned collections of objects. Transformations construct a new RDD from a previous one. Actions compute a result based on an RDD. Multiple computation steps are expressed as directed acyclic graph (DAG). The DAG execution model is a generalization of the Hadoop MapReduce computation model.
